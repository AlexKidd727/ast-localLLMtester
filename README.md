# AST-LocalLLMTester

Разработчик: Студенников Алексей Олегович [https://ast-softpro.ru]

Инструмент для автоматического тестирования и сравнения локальных LLM-моделей через [LM Studio](https://lmstudio.ai/) API. Веб-интерфейс на Flask позволяет запускать тесты, просматривать результаты и оценивать качество генерации кода.

## Возможности

- Подключение к LM Studio по API (совместим с OpenAI API)
- Два типа тестов для каждой модели:
  - **Генерация Python-скрипта** по текстовому описанию (обработка CSV, фильтрация, среднее, экспорт в JSON)
  - **Исправление ошибки** в Python-коде по трассировке ошибки (ZeroDivisionError)
- Автоматическая оценка сгенерированного кода через [OpenRouter](https://openrouter.ai/) (опционально)
- Веб-интерфейс с таблицей результатов, сортировкой, логом выполнения и просмотром оценок
- Сохранение результатов в JSON и отдельных файлах по каждой модели
- Автоматическая выгрузка модели после теста

## Быстрый старт

### Требования

- Python 3.10+
- Запущенный [LM Studio](https://lmstudio.ai/) с загруженной моделью и включённым API-сервером

### Установка

```bash
git clone https://github.com/AlexKidd727/ast-localLLMtester.git
cd ast-localLLMtester

# Создать виртуальное окружение (рекомендуется)
python -m venv venv
venv\Scripts\activate      # Windows
# source venv/bin/activate  # Linux/macOS

# Установить зависимости
pip install -r requirements.txt
```

### Настройка

Скопируйте файл `.env.example` в `.env` и укажите свои значения:

```bash
copy .env.example .env     # Windows
# cp .env.example .env     # Linux/macOS
```

Основные параметры в `.env`:

| Переменная | Описание | По умолчанию |
|---|---|---|
| `LM_STUDIO_BASE_URL` | Адрес API LM Studio | `http://localhost:1234` |
| `OPENROUTER_API_TOKEN` | Токен OpenRouter для оценки кода | _(пусто -- оценка отключена)_ |
| `OPENROUTER_MODEL` | Модель OpenRouter для оценки | `arcee-ai/trinity-large-preview:free` |
| `MAX_TOKENS` | Максимум токенов в ответе | `4096` |
| `TEMPERATURE` | Температура генерации | `0.7` |

### Запуск

```bash
python app.py
```

Откройте в браузере: [http://localhost:5987](http://localhost:5987)

## Как это работает

1. **Выбор модели** -- веб-интерфейс получает список загруженных моделей из LM Studio
2. **Запуск теста** -- для выбранной модели последовательно выполняются два задания:
   - Генерация Python-скрипта по описанию задачи
   - Исправление Python-скрипта с ошибкой по трассировке
3. **Оценка** -- если настроен OpenRouter, сгенерированный код автоматически оценивается внешней LLM по шкале 1--10
4. **Результаты** -- сохраняются в `results.json` и в папке `results/<model_id>/`
5. **Выгрузка** -- после теста модель автоматически выгружается из памяти

## Добавление своих тестов

Тестовые задания определены в `lm_model_tester.py`:

- **Генерация**: измените `PYTHON_GENERATION_PROMPT` для другого задания
- **Исправление ошибки**: замените `test_files/buggy_script.py` и `test_files/error_log.txt` своими файлами
